When to Use

Best applications:

The visitor pattern is most useful when few changes will be made to the class structure, .

Advantages:

  • Follows the Open/Closed Principle
  • Code flexibility: functionality is easily added and removed
  • Decoupling of operations from the object structure
  • Knowledge HTML5 & jQuery
  • Knowledge of responsive web design

Disadvantages:

  • Encapsulation is easily broken
  • For every new subclass, a method must be added to each concrete visitor
  • Code may be complicated and hard to follow
  • An iterator must be implemented to traverse objects
